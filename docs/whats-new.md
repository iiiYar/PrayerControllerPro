# What Changed

## Version 1.0.10

- إصدار build مستقل لميزة `Volume guard` داخل مجلد `releases\v1.0.10\win-x64`
- تثبيت رقم التطبيق والملف التنفيذي على `1.0.10`
- إبقاء طريقة `Play/Pause key` القديمة كخيار، مع توفر `Volume guard` كطريقة اختيارية أكثر أمانًا

## Version 1.0.7

- بناء نظام `Logs` كامل يكتب ملفات يومية داخل `%AppData%\PrayerControllerPro\logs`
- إضافة نافذة `Logs` لعرض آخر أحداث التطبيق وفتح مجلد السجلات ومسحها
- تسجيل أحداث التشغيل، الإعدادات، تحديث الجدول، الأتمتة، التذكيرات، الإشعارات، والأخطاء
- إضافة إعدادات إشعارات داخل `App Settings` تشمل Windows tray notifications وDiscord webhook
- إضافة اختبار Discord من نافذة الإعدادات قبل حفظ الإعدادات
- إعادة تصميم `Widget Mode` جذريًا مع شريط accent جانبي وتركيز أكبر على الصلاة والعد التنازلي

## Version 1.0.8

- إعادة تصميم نافذة `Add/Edit Reminder` لتستخدم حقول فاتحة وواضحة مثل نافذة الإعدادات
- إزالة قسم `Automation` من نافذة التذكير واستبداله بتحكم مباشر في `Pause before` و`Resume after`
- جعل التذكير المخصص يدعم التحكم قبل/بعد الوقت مثل الصلاة المخصصة
- إعادة ترتيب قسم `Notifications` في الإعدادات إلى قنوات إشعار وأحداث قابلة للتنبيه
- تعديل منطق الاستئناف ليكون بعد وقت الإقامة مضافًا إليه `Resume after` بدل حسابه من وقت الأذان

## Version 1.0.9

- إضافة تحميل `Adhan preset` من رابط مرخّص داخل الإعدادات مع حفظ الملف في كاش محلي
- إضافة تحميل `Iqama preset` من رابط مرخّص داخل الإعدادات مع حفظ الملف في كاش محلي
- إضافة أزرار `Download` و`Clear cache` لكل من الأذان والإقامة
- إبقاء خيار الملف المحلي للأذان والإقامة بجانب خيار preset المحمّل
- إضافة إعداد `Media control method` لاختيار الطريقة القديمة `Play/Pause key` أو الطريقة الجديدة `Volume guard`
- إضافة `Volume guard` لخفض صوت تطبيقات الميديا الأخرى أثناء نافذة الصلاة بدون الضغط على زر Play/Pause
- إضافة مستويات `Volume guard` جاهزة `0%` و`3%` و`5%` مع خيار `Custom` من 0 إلى 100

## Architecture

- ترقية التطبيق إلى `.NET 8`
- نقل الواجهة من `WinForms` إلى `WPF`
- تقسيم المشروع إلى:
  - `PrayerControllerPro.Core` لمنطق الأعمال
  - `PrayerControllerPro.App` للواجهة والتكامل مع ويندوز
  - `PrayerControllerPro.Tests` لاختبارات الجدولة

## New Features

- اعتماد ترقيم إصدار رسمي للتطبيق: `1.0.10`
- جعل كل نسخة منشورة تخرج داخل مجلد إصدار مستقل تحت `releases\`

- تحسين نافذة `Add/Edit Reminder` بحقل اسم أوضح واختيار وقت بصيغة `12-hour` مع `AM/PM`
- إضافة تحرير كامل للتذكيرات المخصصة من نفس نافذة `Reminder`
- إضافة خيار أتمتة للتذكير المخصص مع مدة استئناف الوسائط بعد التذكير
- تحسين تموضع حقول نافذة `Reminder` وتنظيم اختيار الوقت بشكل أوضح
- تحسين وضوح حقول نافذة الإعدادات وإظهار المدينة المختارة بشكل صريح
- صقل لوحة التحكم بإزالة التكرار من Hero ونقل زر التذكيرات إلى قسم الجدول
- إزالة الفوتر السفلي ونقل الوقت والتاريخ إلى رأس الواجهة
- إضافة شريط أوامر علوي يضم `Widget` و`Reminder` و`Settings` و`About` و`Logs` و`Exit`
- إعادة ترتيب الهيدر العلوي بحيث يظهر اسم التطبيق والوقت والتاريخ في سطر واحد أخف
- إزالة تكرار المدينة من الهيدر لأن الموقع يظهر بالفعل في شريط معلومات الجدول
- إبقاء زر `Refresh` داخل قسم جدول الصلوات لأنه مرتبط مباشرة بتحديث الجدول
- تحسين شريط معلومات الجدول ليعرض المصدر والطريقة والموقع كبيانات منفصلة
- تصغير `Widget Mode` وإعادة ترتيبه ليعرض الصلاة والعد التنازلي والوقت بشكل أكثر تركيزًا
- توسيع نافذة Dashboard إلى `460px` لتحسين القراءة وتقليل ضغط الصفوف
- زيادة ارتفاع نافذة Dashboard إلى `840px` لعرض مريح أكثر
- إضافة وقت الأذان والإقامة مباشرة داخل Hero الصلاة القادمة
- تحسين شكل شريط التمرير ليتماشى مع الثيم الداكن
- اختصار ملخص الأتمتة داخل صفوف الصلوات لتقليل الزحمة البصرية
- إضافة واجهة WPF جديدة تعرض:
  - الوقت الحالي
  - العد التنازلي
  - قائمة الصلوات والتذكيرات
  - حالة الصلاة النشطة أو القادمة
- إعادة تصميم الواجهة إلى لوحة تحكم طولية أنظف وأسهل في المسح البصري
- تحويل النصوص الظاهرة في التطبيق إلى الإنجليزية فقط
- إضافة `Widget Mode` مصغّر من داخل نفس التطبيق مع نافذة صغيرة `always-on-top`
- إضافة نافذة إعدادات عامة للمدينة وطريقة الحساب وملفات الأذان والإقامة
- توحيد تصميم نوافذ `App Settings` و`Prayer Automation` و`Add/Edit Reminder` مع لغة الواجهة الرئيسية
- إضافة دعم اختيار ملف أذان محلي

- إضافة تذكيرات مخصصة يومية

- دعم حماية صوت اختيارية عبر `Volume guard` بدل الاعتماد الإجباري على زر `Play/Pause`

## Fixes

- إصلاح مشكلة الاعتماد على دولة ثابتة في طلبات API
- إصلاح مشكلة ضياع بعض الصلوات إذا لم يكن التطبيق على الثانية الدقيقة نفسها
- إصلاح مشكلة تكرار ضغط زر الوسائط بعد تحديث الجدول أثناء نافذة صلاة نشطة
- إصلاح تعطل بدء التطبيق عند وجود ملف إعدادات JSON تالف
- إصلاح منطق الاستئناف حتى ينفذ مرة واحدة فقط بعد أن يكون التطبيق هو من أوقف الوسائط
- تقليل خطر تشغيل الميديا بالغلط وقت الأذان عبر خيار `Volume guard` الذي يخفض صوت التطبيقات الأخرى ويعيدها بعد الصلاة
- إصلاح التعامل مع المدن ذات المناطق الزمنية المختلفة
- إضافة Cache يومي لخفض أثر انقطاع الشبكة أو فشل الطلب

## Verification

- `dotnet build PrayerControllerPro.sln`
- `dotnet test tests/PrayerControllerPro.Tests/PrayerControllerPro.Tests.csproj`

## Known Limits

- خيار `Play/Pause` القديم ما زال يعتمد على مفتاح النظام، أما `Volume guard` فيخفف الخطر عبر خفض صوت تطبيقات الميديا الأخرى بدل تبديل التشغيل
- لا توجد ملفات أذان/إقامة مضمّنة داخل المستودع؛ يتم اختيارها من الجهاز محليًا
