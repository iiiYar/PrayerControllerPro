# What Changed

## Version 1.3.0

- إصدار أول GitHub Release رسمي للتطبيق مع ملف ZIP جاهز للتنزيل
- جعل `VolumeGuard` هو الوضع الافتراضي الحقيقي للتثبيت الجديد لتقليل خطر تشغيل الميديا بالغلط
- استرجاع صوت التطبيقات المحمية فورياً عند إغلاق التطبيق بدل انتظار fade غير مضمون قبل خروج العملية
- توسيع نافذة التقاط صوت الأذان والإقامة إلى 5 دقائق بعد السكون أو تشغيل التطبيق المتأخر
- تحسين تغطية الاختبارات لحالة الإقامة المتأخرة بجانب الأذان
- تحديث اعتماديات NuGet الخاصة بالتطبيق والاختبارات
- رفع رقم التطبيق والملف التنفيذي إلى `1.3.0`

## Architecture

- ترقية التطبيق إلى `.NET 8`
- نقل الواجهة من `WinForms` إلى `WPF`
- تقسيم المشروع إلى:
  - `PrayerControllerPro.Core` لمنطق الأعمال
  - `PrayerControllerPro.App` للواجهة والتكامل مع ويندوز
  - `PrayerControllerPro.Tests` لاختبارات الجدولة

- إصلاح مشكلة الاعتماد على دولة ثابتة في طلبات API
- إصلاح مشكلة ضياع بعض الصلوات إذا لم يكن التطبيق على الثانية الدقيقة نفسها
- إصلاح مشكلة تكرار ضغط زر الوسائط بعد تحديث الجدول أثناء نافذة صلاة نشطة
- إصلاح تعطل بدء التطبيق عند وجود ملف إعدادات JSON تالف
- إصلاح منطق الاستئناف حتى ينفذ مرة واحدة فقط بعد أن يكون التطبيق هو من أوقف الوسائط
- تقليل خطر تشغيل الميديا بالغلط وقت الأذان عبر خيار `Volume guard` الذي يخفض صوت التطبيقات الأخرى ويعيدها بعد الصلاة
- إصلاح التعامل مع المدن ذات المناطق الزمنية المختلفة
- إضافة Cache يومي لخفض أثر انقطاع الشبكة أو فشل الطلب

## Known Limits

- خيار `Play/Pause` ما زال يعتمد على مفتاح النظام، أما `Volume guard` فيخفف الخطر عبر خفض صوت تطبيقات الميديا الأخرى بدل تبديل التشغيل
- لا توجد ملفات أذان/إقامة مضمّنة داخل المستودع؛ يتم اختيارها من الجهاز محليًا
